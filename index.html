Q.2 (a) What is the difference between stateless and stateful firewalls? 03 Marks

Answer:

Stateless and Stateful Firewalls differ significantly in their approach to network traffic filtering:

Stateless Firewall:

  • Filters packets based on individual packet information
  • No memory of previous connections
  • Filters based on source/destination IP, port numbers
  • Faster processing but less secure
  • Cannot detect complex attack patterns
  • Example: Basic ACLs on routers

Stateful Firewall:

  • Maintains state of active connections
  • Tracks connection state from start to finish
  • Makes decisions based on connection context
  • More secure but requires more resources
  • Can detect and block suspicious patterns
  • Example: Modern NGFWs, ASA firewalls

Stateful firewalls provide better security by understanding the context of network traffic, while stateless firewalls offer simpler, faster filtering for basic security needs.

Q.2 (c) Evaluate the role of Network Address Translation (NAT) in enhancing network security. 07 Marks

Answer:

Network Address Translation (NAT) plays a significant role in network security through multiple mechanisms:

1. IP Address Hiding

NAT conceals internal IP addresses from external networks:

  • Private IP addresses remain hidden behind public IP
  • External attackers cannot directly access internal hosts
  • Reduces the attack surface significantly
  • Makes network reconnaissance more difficult

2. Stateful Inspection

  • NAT maintains connection state tables
  • Only allows return traffic for established connections
  • Blocks unsolicited inbound connections
  • Prevents many types of network attacks

3. Port Randomization

  • Randomizes source ports for outbound connections
  • Makes port scanning attacks less effective
  • Adds unpredictability to network communications

4. Security Benefits

  • Network Segmentation: Creates logical boundaries between networks
  • Access Control: Implicitly controls inbound traffic
  • Logging Capabilities: Tracks all address translations
  • Load Distribution: Can distribute traffic across multiple servers

5. Limitations and Considerations

  • Not a replacement for proper firewalls
  • Can interfere with some applications and protocols
  • May complicate VPN configurations
  • Doesn't protect against all attack types

6. Best Practices

  • Combine NAT with proper firewall rules
  • Implement strict outbound filtering
  • Regular monitoring of NAT tables
  • Use static NAT for critical services

Conclusion: While NAT provides significant security benefits through address hiding and stateful inspection, it should be considered as part of a comprehensive security strategy rather than a standalone solution. When properly implemented, NAT significantly enhances network security by creating a barrier between internal and external networks.

1. Virus

  • Spread: Requires human action and host file
  • Behavior: Attaches to legitimate files
  • Replication: Copies itself when host file executes
  • Example: File infector viruses, macro viruses

2. Trojan

  • Spread: Disguised as legitimate software
  • Behavior: Does not self-replicate
  • Action: Performs malicious activities hidden from user
  • Example: Remote access Trojans, banking Trojans

3. Worm

  • Spread: Self-replicates across networks
  • Behavior: Exploits vulnerabilities automatically
  • Replication: No human intervention required
  • Example: Code Red, SQL Slammer

  1. Program allocates fixed-size buffer
  2. Input exceeds buffer capacity
  3. Data overflows into adjacent memory
  4. Overwrites critical data (return addresses)
  5. Program flow redirected to malicious code

Types:

  • Stack-based overflow
  • Heap-based overflow
  • Integer overflow

Prevention:

  • Input validation
  • Safe functions
  • ASLR and DEP
  • Regular patching

Buffer overflows can lead to arbitrary code execution and system compromise.

DoS (Denial of Service) Attack

  • Single source attack
  • One computer/system
  • Easier to trace
  • Limited bandwidth
  • Simple to execute

DDoS (Distributed Denial of Service) Attack

  • Multiple sources
  • Botnet of compromised systems
  • Difficult to trace
  • Massive bandwidth
  • Complex coordination

Key Differences:

  • DoS: Single attacker, easier to block
  • DDoS: Multiple attackers, harder to mitigate
  • DDoS more destructive
  • DDoS requires botnet infrastructure

Both aim to make services unavailable but differ in scale and complexity.
